# 136 · Desktop 右栏信息流节奏与权重优化（Phase 1.1）

## 背景与问题

基于 Phase 1 右栏重构，当前交互可用，但仍存在明显的“模块堆叠感”：

1. **纵向节奏过碎**：header / capture / stream / frozen / footer 都是独立卡片边框，视觉切割过多。
2. **信息权重不清**：候选区标题与卡片主内容抢权重，用户难以形成自然浏览路径。
3. **操作语气过工具化**：`详情 / 标记` 偏后台审批语境，不符合温和产品气质。
4. **流感不足**：当前更像功能区堆叠，而非自然向下的信息流。

## 优化目标

在不引入真实数据的前提下，将右栏由“模块列表”调整为“流式阅读面板”：

- 视觉上减少一级边框，强化卡片流
- 交互上保留互斥手风琴，但让冻结区更像抽屉
- 语义上弱化栏目标题，增强内容本体优先级
- 底部摘要进一步极简，不抢主视觉

## 设计与实现范围

### A. 候选区标题内嵌化（弱栏目感）

- 去掉“候选区（实时）”独立块标题卡
- 在候选流顶部改为小字说明：`最近整理结果`
- 标题作为信息流内联说明，不再占据独立分组视觉

### B. 视觉节奏收敛（减少容器边框）

- 一级容器尽量用留白分隔，不再层层加框
- 保留明显边框的核心单元仅限：
  - Capture 区
  - CandidateCard
- 卡片边框减淡，适当使用背景层次或轻阴影做分离

### C. 底部摘要极简化

- 从独立块改为单行弱信息文本
- 文案统一为：`待确认 x · 已采纳 y · heartbeat t`
- 固定贴底，不参与中部滚动，不可溢出

### D. 冻结区抽屉化

- 冻结区默认单行头部（低高度）
- 展开时像抽屉滑出；收起时仅保留一行入口
- 与候选区保持互斥展开（任何时刻默认候选区打开）

### E. 按钮语义轻量化

- `详情 / 标记` 调整为更轻文案：
  - `采纳`
  - `编辑`
  - `暂存`
- 可选：默认弱显示，仅 hover 时增强可见度

## 验收标准

- 右栏整体呈现连续信息流，一级分块感显著下降
- 用户首屏视线优先落在候选卡片内容，而非栏目标题
- 冻结区关闭时仅占一行，展开时为抽屉式体验
- 底部摘要固定在最底且弱化呈现，不溢出
- 操作按钮语言与视觉更加温和，非后台审批风格
