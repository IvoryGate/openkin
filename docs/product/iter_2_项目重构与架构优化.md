## 3. 功能模块

### 3.1 Agent管理模块

**功能描述**：用户可以创建、查看、删除智能Agent，每个Agent拥有独立的配置和性格设定。

**核心功能**：

#### 创建Agent
- **入口**：引导流程（首次使用） / 设置页面（后续创建）
- **模板选择**：
  - 🤖 通用助手 - 全能型助手，适合日常问答
  - 💻 技术专家 - 专注技术问题，擅长代码和架构
  - ✍️ 写作助手 - 创意写作、文案优化和内容生成
- **自定义配置**：
  - Agent名称（必填）
  - Agent角色（可自定义或选择模板默认值）
  - Agent描述（可自定义或选择模板默认值）
  - System Prompt（可自定义或选择模板默认值）

**数据持久化**：
- Agent配置存储在`~/.openkin/agents/{agentId}/meta.json`
- Agent的Soul.md存储在`~/.openkin/agents/{agentId}/soul.md`

#### 查看Agent列表
- **入口**：侧边栏Agent列表
- **显示内容**：
  - Agent名称
  - Agent角色
  - Agent描述
  - 创建时间
- **交互**：
  - 点击切换当前Agent
  - 右键菜单（删除Agent）

#### 删除Agent
- **触发条件**：用户确认删除
- **删除内容**：
  - Agent配置文件（meta.json）
  - Agent的Soul.md文件
  - Agent的聊天记录（如果有）

**安全措施**：
- 二次确认对话框
- 删除后不可恢复

---

### 3.2 聊天对话模块

**功能描述**：提供实时流式对话体验，支持多轮对话和历史记录。

**核心功能**：

#### 消息发送
- **输入方式**：
  - 文本输入框（支持多行）
  - 快捷键支持（Enter发送，Shift+Enter换行）
- **输入限制**：
  - 最大字符数：4000
  - 自动高度调整（1-10行）

#### 流式响应
- **响应方式**：WebSocket流式推送
- **渲染效果**：
  - 逐字显示（打字机效果）
  - 自动滚动到底部
  - Markdown渲染（待实现）

#### 消息显示
- **用户消息**：
  - 样式：右对齐，蓝色背景
  - 显示：用户头像 + 消息内容 + 时间戳
- **AI消息**：
  - 样式：左对齐，灰色背景
  - 显示：Agent头像 + 消息内容 + 时间戳
  - 支持流式追加动画

#### 历史记录
- **存储位置**：`~/.openkin/memories/{agentId}/{sessionId}.json`
- **记录内容**：
  - 消息列表（role + content）
  - 会话元数据（创建时间、最后更新时间）
- **上下文窗口**：
  - 默认保留最近20条消息
  - 支持配置调整

---

### 3.3 配置管理模块

**功能描述**：管理用户的API密钥和应用配置。

**核心功能**：

#### API密钥管理
- **支持的提供商**：
  - OpenAI（GPT系列模型）
  - Anthropic（Claude系列模型）
- **密钥输入**：
  - 脱敏显示（sk-****...）
  - 显示/隐藏切换
  - 实时格式验证
- **密钥验证**：
  - 发送最小测试请求
  - 验证密钥有效性
  - 验证模型可用性
  - 返回验证结果（成功/失败 + 错误信息）

#### 自定义配置
- **自定义Endpoint**：
  - 支持自定义API endpoint
  - 适用于本地模型服务或第三方服务
  - 格式验证
- **自定义模型**：
  - 支持自定义模型名称
  - 适用于非标准模型名称

#### 安全存储
- **加密方式**：AES-256-GCM
- **密钥派生**：基于主机指纹（hostname + username）
- **存储位置**：`~/.openkin/config.json`

### 3.4 用户引导模块

**功能描述**：首次使用时的引导流程，帮助用户快速上手。

**引导步骤**：

#### 第一步：欢迎页面
**内容**：
- 产品介绍
- 核心功能说明
- 使用场景展示
- 开始引导按钮

#### 第二步：配置API
**内容**：
- 选择LLM提供商（OpenAI/Anthropic）
- 输入API密钥
- 验证密钥有效性
- （可选）配置自定义endpoint

#### 第三步：创建Agent
**内容**：
- 选择Agent模板（通用/技术/写作）
- 输入Agent名称
- （可选）自定义Agent角色和描述
- 创建Agent

#### 第四步：完成引导
**内容**：
- 引导完成确认
- 功能快速入门指南
- 进入主界面按钮

**引导状态**：
- 存储位置：`config.json`中的`initialized`字段
- 首次启动后显示引导流程
- 完成后不再显示

## 5. 交互设计

### 5.1 聊天交互流程

```
用户输入消息
    ↓
[Enter键]
    ↓
显示用户消息（右侧）
    ↓
发送到后端（WebSocket）
    ↓
[等待响应]
    ↓
创建AI消息气泡（左侧）
    ↓
流式接收内容
    ↓
逐字显示（打字机效果）
    ↓
[响应完成]
    ↓
显示AI消息完成
```

### 5.2 Agent切换流程

```
用户点击侧边栏Agent
    ↓
高亮当前Agent
    ↓
加载Agent的聊天记录
    ↓
显示历史消息
    ↓
准备接收新消息
```

### 5.3 引导流程

```
首次启动应用
    ↓
[检查config.json]
    ↓
initialized = false?
    ↓ YES
显示引导流程
    ├─ 步骤1: 欢迎
    ├─ 步骤2: 配置API
    ├─ 步骤3: 创建Agent
    └─ 步骤4: 完成
    ↓
更新config.json (initialized = true)
    ↓
进入主界面
    ↓ NO
直接进入主界面
```

## 6. 数据管理

### 6.1 数据存储结构

```
~/.openkin/
├── config.json              # 应用配置
├── agents/                  # Agent数据
│   ├── {agentId}/
│   │   ├── meta.json       # Agent元数据
│   │   └── soul.md         # Agent的Soul文件
├── memories/               # 聊天记录
│   └── {agentId}/
│       └── {sessionId}.json # 会话记录
├── souls/                  # 共享Soul文件
└── skills/                 # 共享技能文件
```

### 6.2 数据安全

- **配置文件加密**：AES-256-GCM加密API密钥
- **本地存储**：所有数据存储在用户主目录
- **隐私保护**：不发送任何数据到外部服务器（除了LLM API调用）
- **跨平台兼容**：路径处理支持三大平台

## 7. 性能要求

### 7.1 响应时间

- **消息发送**：< 100ms

- **页面切换**：< 100ms

- **内存占用**：< 500MB（空闲时）
- **CPU占用**：< 10%（空闲时）

- **帧率**：30-60 FPS
- **延迟**：< 50ms（首字到显示）
