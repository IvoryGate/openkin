# 产品文档 - 迭代一：基础框架搭建

**迭代轮数**：1  
**迭代主题**：基础框架搭建（MVP）  
**预估周期**：1~2 周  
**状态**：设计中

---

## 1. 迭代目标

建立 OpenKin 多 Agent 智能协作系统的最小可行产品（MVP），实现：

1. 应用可正常启动，呈现开机引导界面
2. 用户完成首个 Agent 创建（含 API 密钥配置）
3. Soul.md 个性配置文件系统可读写
4. Agent 能够响应基础对话（单轮/多轮）

---

## 2. 用户故事

### US-01 首次启动引导

**角色**：新用户  
**期望**：首次打开应用时，看到友好的引导界面，逐步完成初始化配置  
**价值**：降低上手门槛，确保系统在第一次使用时就能正常工作

**验收条件**：
- [ ] 检测到无任何 Agent 配置时，自动进入引导流程
- [ ] 引导流程分步骤展示（进度指示）
- [ ] 用户可以跳过非必填步骤，但 API 密钥至少需配置一项
- [ ] 完成后进入主控制台

---

### US-02 API 密钥配置

**角色**：新用户  
**期望**：能够输入并验证 AI 模型 API 密钥（OpenAI / Anthropic / 自定义端点）  
**价值**：确保 Agent 具备调用 AI 能力

**验收条件**：
- [ ] 支持输入 OpenAI API Key（必选，默认）
- [ ] 支持输入 Anthropic API Key（可选）
- [ ] 支持输入自定义 LLM 端点 URL（可选）
- [ ] 提交后进行连通性验证，给出成功/失败提示
- [ ] API Key 以加密方式持久化存储于本地

---

### US-03 创建首个 Agent

**角色**：新用户  
**期望**：通过向导界面创建第一个 Agent，设定其名称、角色描述和技能  
**价值**：让用户立刻体验 Agent 个性化能力

**验收条件**：
- [ ] 提供 Agent 名称输入框（必填，2~50 字符）
- [ ] 提供角色描述输入框（选填，最多 500 字符）
- [ ] 支持从预置模板中选择（如：通用助手、技术专家、写作助手）
- [ ] 创建后生成对应的 Soul.md 文件并写入配置目录
- [ ] 创建完成后自动进入与该 Agent 的对话界面

---

### US-04 与 Agent 基础对话

**角色**：已创建 Agent 的用户  
**期望**：能够在主界面与 Agent 进行文字对话，Agent 能给出有意义的回复  
**价值**：验证整个系统闭环可运行

**验收条件**：
- [ ] 提供消息输入框，支持回车发送或点击发送按钮
- [ ] 消息以气泡形式展示，区分用户/Agent 消息
- [ ] Agent 响应时间 < 10 秒（网络正常情况）
- [ ] Agent 回复内容受其 Soul.md 中角色设定影响
- [ ] 支持会话历史滚动查看
- [ ] 支持 Markdown 格式渲染

---

### US-05 Soul.md 编辑

**角色**：已创建 Agent 的用户  
**期望**：能够查看并编辑 Agent 的 Soul.md 配置文件  
**价值**：允许用户自定义 Agent 人格和行为

**验收条件**：
- [ ] 提供 Soul.md 的 GUI 编辑界面（表单形式）
- [ ] 编辑后实时保存到本地文件
- [ ] 修改后的配置在下一次对话中生效
- [ ] 支持恢复默认配置

## 3. 功能范围

### 3.1 本迭代包含（In Scope）

| 功能模块 | 功能描述 |
|---------|---------|
| 引导界面 | 首次启动检测、分步引导流程、完成跳转 |
| API 密钥管理 | 密钥输入、连通性验证、加密存储 |
| Agent 创建向导 | 名称/描述/模板选择、Soul.md 生成 |
| Soul.md 系统 | 文件格式定义、GUI 编辑、读写接口 |
| 基础对话界面 | 消息收发、历史展示、Markdown 渲染 |
| Agent 引擎（基础版） | 单 Agent 对话、System Prompt 注入、多轮上下文 |

### 3.2 本迭代不包含（Out of Scope）

- 多 Agent 协作
- 任务调度系统
- 记忆归档（长期记忆）
- 定时巡检功能
- 平台桥接（Telegram/Discord/微信）
- 技能市场

## 4. 界面与交互设计

### 4.1 页面流转

```
[应用启动]
    │
    ▼
[检测是否首次启动?]
    │
    ├─ 是 ──► [引导页 Step 1: 欢迎页]
    │             │
    │         [Step 2: API 密钥配置]
    │             │
    │         [Step 3: 创建首个 Agent]
    │             │
    │         [Step 4: 完成 → 进入主界面]
    │
    └─ 否 ──► [主界面（对话界面）]
```

### 4.2 主界面布局（桌面应用）

```
┌─────────────────────────────────────────────────────────┐
│  ┌──────────────┐  ┌─────────────────────────────────┐  │
│  │  侧边栏       │  │  对话区域                        │  │
│  │              │  │                                  │  │
│  │  [Agent 列表] │  │  ┌─────────────────────────┐   │  │
│  │  > 技术助手 ✓│  │  │  [Agent 消息气泡]        │   │  │
│  │    写作助手  │  │  │  [用户消息气泡]          │   │  │
│  │              │  │  │  [Agent 消息气泡]        │   │  │
│  │  [+新建Agent]│  │  └─────────────────────────┘   │  │
│  │              │  │                                  │  │
│  │  [⚙️ 设置]   │  │  ┌─────────────────────────┐   │  │
│  └──────────────┘  │  │  输入框 [发送]           │   │  │
│                    │  └─────────────────────────┘   │  │
│                    └─────────────────────────────────┘  │
└─────────────────────────────────────────────────────────┘
```

### 4.3 引导流程设计

**Step 1 - 欢迎页**
- 产品 Logo + 产品名称 "OpenKin"
- 简短的一句话介绍
- [开始配置] 按钮

**Step 2 - API 密钥配置**
- OpenAI API Key 输入框（带密码掩码）
- [验证连接] 按钮，实时反馈
- 支持跳过，但跳过后提示功能受限
- 支持 Anthropic / 自定义端点的折叠面板

**Step 3 - 创建首个 Agent**
- Agent 名称输入框（placeholder: "如：我的助手"）
- 角色描述文本域（placeholder: "描述这个 Agent 擅长做什么..."）
- 预置模板卡片列表（3 种：通用/技术/写作）
- [创建并开始] 按钮

**Step 4 - 完成**
- 成功动画
- 简介一下 Agent 已就绪
- [开始对话] 按钮 → 进入主界面

## 5. Soul.md 格式规范（第一阶段定义）

```markdown
# Agent 个性配置

## 基本信息
- 名称: {agent_name}
- 角色: {agent_role}
- 描述: {agent_description}
- 创建时间: {created_at}

## 人格与风格
- 沟通风格: {style}        # 如：专业严谨 / 亲切友好 / 简洁高效
- 工作方式: {work_style}   # 如：分步解析 / 直接答案 / 互动引导
- 语言偏好: {language}     # 如：中文为主

## 核心能力
{capabilities_list}

## 系统提示词（System Prompt）
{system_prompt}
```

## 6. 数据存储设计

### 6.1 本地目录结构

```
~/.openkin/
├── config.json          # 全局配置（含加密 API Key）
├── agents/
│   ├── {agent_id}/
│   │   ├── soul.md      # Agent 个性配置
│   │   └── meta.json    # Agent 元数据（id、创建时间等）
├── sessions/
│   └── {agent_id}/
│       └── {session_id}.json   # 会话历史
```

### 6.2 config.json 结构

```json
{
  "version": "1.0",
  "initialized": true,
  "active_agent_id": "uuid",
  "api_keys": {
    "openai": "<encrypted>",
    "anthropic": "<encrypted>",
    "custom_endpoint": ""
  },
  "ui": {
    "theme": "dark",
    "language": "zh-CN"
  }
}
```

## 7. 非功能性需求

| 指标 | 要求 |
|------|------|
| 首次启动时间 | < 3 秒 |
| Agent 响应延迟 | < 10 秒（含 API 调用） |
| 内存占用（空闲） | < 300 MB |
| 支持操作系统 | macOS 13+、Windows 10+、Linux（Ubuntu 22+） |
| 数据安全 | API Key 使用 AES-256 加密存储，不上传云端 |
