# v2 模型工作模式

## 一、模式总览

v2 区分三种工作模式：

### 1.1 `high-capability mode`

适用场景：
- 架构设计
- 跨层 contract 收口
- 多方案权衡
- 高风险边界调整

特点：
- 可以做方向判断
- 可以定义新规则
- 可以规划较大阶段
- 可以冻结弱模型后续执行所需的单一路径决策

### 1.2 `budget mode`

适用场景：
- 低预算阶段
- 能力较弱模型
- 以稳定推进为主

特点：
- 只能在现有规则和计划内执行
- 以小步实现、修复、验证为主
- 遇到边界问题立即停下

### 1.3 `maintenance mode`

适用场景：
- 文档整理
- 链接修复
- 归档移动
- 脚本更新

## 二、v2 升级条件

任何模型在执行中遇到以下情况，必须停止并汇报：

1. 需要修改已冻结的跨层 contract
2. 需要调整架构边界
3. 连续两轮无法通过 CI 验证
4. 无法明确定位应修改的文件
5. 需要在多个方案之间做设计取舍
